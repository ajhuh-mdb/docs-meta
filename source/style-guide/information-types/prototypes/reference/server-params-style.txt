Details to Document
-------------------

When documenting a new server parameter, include the following details 
in addition to specific information about the new parameter:

.. list-table::
   :widths: 40 40
   :header-rows: 1

   * - Documentation Detail
     - Necessity
     - Description

   * - mongod / mongos Support
     - Required
     - Whether the parameter is supported on ``mongod``, ``mongos``, or both.
       Use ``|mongod-only|``, ``|mongos-only|``, or ``|both|`` as appropriate.

   * - Version the Parameter was Added or Changed In
     - Optional
     - 
       - For recent versions, include the version the parameter was added 
         or changed in.
       - For archived versions, delete the version added tag.

   * - OS Availability
     - Optional
     - Some parameters are only available on certain operating systems. If the 
       parameter is only supported on certain operating systems, specify which 
       it's supported on.

   * - Argument Type
     - Required
     - The type of argument the parameter expects: integer, string, boolean,
       document, etc.

   * - Default Value
     - Required
     - Specify the default value the parameter takes.
   
   * - Startup and Runtime Support
     - Required
     - For details, see :ref:`startup-runtime-support`.

.. _startup-runtime-support:

Startup and Runtime Support
~~~~~~~~~~~~~~~~~~~~~~~~~~~

Whether the parameter is supported at startup, runtime, or both.

You can use the provided includes to document startup / runtime support.

To specify the parameter is only supported at startup, use:

.. code-block:: none

   .. include:: /includes/fact-startup-parameter  

To specify the parameter is only supported at runtime, use:

.. code-block:: none

   .. include:: /includes/fact-runtime-parameter  

To specify the parameter is supported both at startup and during
runtime, use:

.. code-block:: none

   .. include:: /includes/fact-runtime-startup-parameter
