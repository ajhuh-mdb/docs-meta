Details to Document
-------------------

When documenting a new server parameter, include the following details 
in addition to specific information about the new parameter:

Whether the parameter is supported on ``mongod``, ``mongos``, or both 
  Use ``|mongod-only|``, ``|mongos-only|``, or ``|both|`` as 
  appropriate.

Added/Changed-in version (if applicable)
  - Include the version the parameter was added or changed in. 
  - Delete the version added tag for archived versions.

OS Availability (if applicable)
  Some parameters are only available on certain operating systems. 
  If the parameter is only supported on certain operating systems, 
  specify which it's supported on.

Type of argument the parameter expects 
  - integer
  - string
  - boolean 
  - document 
  - etc.

Default value
  Specify the default value.

Whether the parameter is supported at startup, runtime, or both
  You can use the provided includes to document startup / runtime support.

  To specify the parameter is only supported at startup, use:

  .. code-block:: none

     .. include:: /includes/fact-startup-parameter  

  To specify the parameter is only supported at runtime, use:

  .. code-block:: none

     .. include:: /includes/fact-runtime-parameter  
 
  To specify the parameter is supported both at startup and during
  runtime, use:

  .. code-block:: none

     .. include:: /includes/fact-runtime-startup-parameter

Example
-------

``tlsUseSystemCA`` documents:

- ``mongod`` support only
- Parameter type
- Default value
- ``tlsUseSystemCA`` is available at startup, but not runtime
